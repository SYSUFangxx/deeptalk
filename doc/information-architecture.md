# 夫妻深夜话题生成器 MVP 信息架构与页面原型说明

## 1. 文档目标

定义 MVP 的页面结构、页面职责、主要模块和关键交互，确保后续设计和开发对齐。

## 2. 产品信息架构

MVP 建议采用 2 个核心页面：

1. 首页
定位：打开即用，快速生成和消费话题。
核心任务：看题、换题、跳过。

2. 分类页
定位：按聊天方向找题。
核心任务：选择分类、浏览该分类话题、继续换题。

## 3. 全局导航结构

建议使用简化底部导航，适合手机单手操作：

1. 首页
2. 分类

说明：

1. 底部导航固定显示。
2. 首页作为默认入口。
3. 分类页作为辅助入口，承接“今晚想聊哪一类”的主动选择。

## 4. 页面原型说明

### 4.1 首页

页面目标：
让用户在最短路径内看到并使用一个话题。

页面模块：

1. 顶部标题区
   - 标题：今晚聊点什么
   - 副标题：给你们一个适合今晚的聊天切口

2. 话题卡片区
   - 分类标签
   - 话题正文
   - 可选引导语
   - 可选辅助标签，如“轻松”“回忆”“未来”

3. 操作区
   - 换一个
   - 跳过

4. 快捷入口区
   - 去分类挑一类

交互说明：

1. 首次进入时自动展示 1 条话题。
2. 点击“换一个”后展示下一条话题。
3. 点击“跳过”后立即切题，并在当前会话中尽量避免短时间重复。

页面低保真结构示意：

```text
[今晚聊点什么]
给你们一个适合今晚的聊天切口

[共同回忆]
结婚以后，你觉得我们之间最珍贵的变化是什么？
如果愿意，也可以顺着讲一个具体瞬间。

[换一个] [跳过]

去分类挑一类
```

### 4.2 分类页

页面目标：
让用户按当下心情或场景主动选方向。

页面模块：

1. 顶部标题区
   - 标题：选个方向聊
   - 副标题：今晚想从哪一类开始

2. 分类列表区
   - 共同回忆
   - 日常相处
   - 未来规划
   - 价值观交流
   - 兴趣与生活
   - 轻松脑洞

3. 分类话题展示区
   - 进入分类后显示当前分类名称
   - 展示 1 条该分类话题
   - 保留换题、跳过能力

交互说明：

1. 用户先点分类。
2. 页面进入该分类详情态。
3. 系统从该分类内展示话题。
4. 用户可在该分类下持续换题。

页面低保真结构示意：

```text
[选个方向聊]
今晚想从哪一类开始

[共同回忆]
[日常相处]
[未来规划]
[价值观交流]
[兴趣与生活]
[轻松脑洞]

-- 点击某一类后 --

[未来规划]
你最期待我们未来五年一起完成的一件事是什么？

[换一个] [跳过]
```

## 5. 核心交互原则

1. 首屏必须直接给结果，不先要求选择。
2. 一次只突出一个话题，避免信息噪音。
3. 所有按钮文案都尽量口语化、低压力。
4. 手机端按钮尺寸要足够大，适合睡前操作。
5. 不做复杂跳转，尽量单层完成主要任务。

## 6. 页面状态说明

建议至少覆盖这些状态：

1. 首页初始加载状态
2. 话题正常展示状态
3. 分类列表状态
4. 分类题目展示状态
5. 题目切换状态

## 7. 设计结论

这一版 MVP 的页面结构应尽量轻，优先保证：

1. 打开就能看到题
2. 点一下就能换题
3. 分类选择足够直观
4. 手机上操作顺手
