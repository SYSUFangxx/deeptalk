# **夫妻深夜话题生成器项目文档**

---

## **项目简介**

### **项目目标**
创建一个能够随机生成高质量夫妻夜聊话题的系统，帮助夫妻/情侣在睡前进行深度、有趣的交流，有效增进感情、增强亲密感、保持关系新鲜度。

### **核心设计理念**
1. **质量优于数量** - 每个话题都应激发思考、促进交流
2. **个性化扩展** - 提供方法论而非固定列表，鼓励用户创造专属话题
3. **情感智能** - 话题覆盖关系各个阶段和维度
4. **易用性** - 生成的话题应即拿即用，无需额外准备

### **技术实现思路**
1. 建立基础话题数据库（种子库）
2. 实现话题分类和标签系统
3. 开发话题生成算法（基于方法论）
4. 设计随机抽取和智能推荐逻辑
5. 增加用户反馈和收藏功能
6. 考虑个性化设置（关系阶段、兴趣偏好等）

## **第二部分：话题衍生方法论（如何从100到1000+）**

### **方法论框架概览**

本系统采用四级话题衍生体系，确保话题无限扩展：

```
基础种子（100个）
    ↓
衍生方法（4大引擎）
    ↓
维度扩展（6个方向）
    ↓
个性化适配 → 无限话题
```

### **四大话题生成引擎**

#### **引擎一：维度替换法**

**核心逻辑**：保持话题句式结构，替换关键维度

**可替换维度库**：
1. **时间维度**：第一次/最后一次/最难忘的一次/十年后...
2. **人物维度**：我/你/我们/父母/孩子/朋友...
3. **空间维度**：家里/旅行中/工作场所/故乡...
4. **事件维度**：吵架时/庆祝时/困难时/无聊时...
5. **感官维度**：看到/听到/闻到/尝到/触到...

**应用示例**：
- 原话题：#61 以后有孩子想起什么小名？
- 维度替换：
  - 时间替换：等我们退休了，想给对方起什么昵称？
  - 人物替换：如果养宠物，会给它起什么名字？
  - 事件替换：一起做的第一个手工作品，想起什么名字？

#### **引擎二：场景具象法**

**核心逻辑**：为抽象话题添加具体、有趣或离奇的假设场景

**场景类型库**：
1. **奇幻场景**：魔法/超能力/时空旅行/外星人...
2. **限制场景**：只有24小时/只有100元/不能说话...
3. **极端场景**：世界末日/获得永生/失去记忆...
4. **反转场景**：身份互换/性别互换/年龄互换...
5. **任务场景**：必须完成某个挑战/必须保守某个秘密...

**应用示例**：
- 原话题：#71 攒够500万后想过哪种躺平生活？
- 场景具象：
  - 限制型：如果这500万必须在24小时内花完，且不能买奢侈品？
  - 奇幻型：如果这500万是完成魔法任务后巨龙给的报酬？
  - 反转型：如果先有500万，但要失去现在的记忆才能花？

#### **引擎三：反向/极端提问法**

**核心逻辑**：从原话题的反面、侧面或推向极端

**反向类型**：
1. **完全反向**：将肯定变否定，积极变消极
2. **程度极端**：将普通情况推到极致
3. **视角反转**：从对方视角、旁观者视角提问
4. **时间反转**：从现在看过去，从未来看现在
5. **价值反转**：交换双方重视的价值取向

**应用示例**：
- 原话题：#33 吵架时最伤人的话是什么？
- 反向提问：
  - 完全反向：吵架时，哪句话反而让你想笑？
  - 极端提问：如果可以合法说一句绝对伤人的话，你会说吗？
  - 视角反转：你觉得我吵架时最怕听到你说什么？

#### **引擎四：生命周期与细节深挖法**

**核心逻辑**：围绕关系发展的时间线和具体细节展开

**关系生命周期阶段**：
```
相识前期 → 暧昧期 → 确定关系 → 热恋期 → 稳定期 → 
磨合期 → 承诺期 → 新婚期 → 家庭建设期 → 中年期 → 
空巢期 → 老年期 → 回顾期
```

**细节挖掘方向**：
1. **第一次系列**：第一次XX时，你在想什么？
2. **最XX系列**：最XX的时刻/决定/礼物...
3. **日常系列**：平时的/经常的/习惯性的...
4. **特殊系列**：节日的/纪念日的/危机的...
5. **变化系列**：从XX到XX的变化...

**应用示例**：
- 针对"第一次"深挖：
  - 第一次见我父母时，最紧张的瞬间是什么？
  - 第一次一起过夜，你偷偷观察到了什么？
  - 第一次收到我的礼物，你当时的真实想法是？

### **六维扩展模型**

将这四大引擎应用于六个扩展维度，形成话题矩阵：

```
          维度替换  场景具象  反向提问  细节深挖
情感维度    ✓        ✓        ✓        ✓
时间维度    ✓        ✓        ✓        ✓
空间维度    ✓        ✓        ✓        ✓
人物维度    ✓        ✓        ✓        ✓
事件维度    ✓        ✓        ✓        ✓
价值维度    ✓        ✓        ✓        ✓
```

**计算示例**：
100个种子话题 × 4种引擎 × 6个维度 × 3种变体 = 7200个潜在话题

### **话题质量评估标准**

在衍生话题时，应确保新话题符合以下标准：

1. **开放性**：不能简单用"是/否"回答
2. **情感价值**：能促进情感连接和深度了解
3. **安全性**：不触及真正的伤疤或禁区
4. **适当深度**：符合当前关系阶段和对话氛围
5. **趣味性**：能激发回答和继续对话的欲望
6. **平衡性**：积极话题与探索性话题保持平衡

## **第三部分：话题生成算法设计**

### **算法核心逻辑**

```
话题生成算法：
1. 确定当前场景和需求
2. 从基础库中选择合适的话题类别
3. 应用话题衍生引擎
4. 根据个性化设置调整
5. 进行质量过滤
6. 输出最终话题
```

### **个性化设置参数**

在项目实现中，应考虑以下个性化参数：

```yaml
# 用户个性化配置文件示例
user_profile:
  relationship_stage: ["热恋期", "稳定期", "已婚有子"]  # 关系阶段
  conversation_mood: ["轻松", "深度", "浪漫", "探索"]   # 对话氛围
  topic_preferences: ["回忆", "未来", "日常", "脑洞"]  # 话题偏好
  avoid_topics: ["前任", "财务", "原生家庭创伤"]       # 避免话题
  recent_topics: ["最近3天聊过的话题ID"]               # 近期话题
  favorite_topics: ["用户收藏的话题ID"]                 # 收藏话题
  conversation_history: ["历史话题及反馈评分"]           # 对话历史
```

### **智能推荐策略**

1. **基于关系的渐进式推荐**：
   - 初期关系：多回忆、兴趣类（占比40%）
   - 稳定期：多未来规划、价值观类（占比30%）
   - 长期关系：多深度反思、重新发现类（占比30%）

2. **基于时间的动态调整**：
   - 工作日晚上：轻松话题为主（70%轻松，30%深度）
   - 周末/假期：深度话题可增加（50%轻松，50%深度）
   - 特殊日期（纪念日等）：回忆和感恩类话题为主

3. **基于反馈的个性化学习**：
   - 记录每个话题的对话时长
   - 记录用户收藏和跳过的话题
   - 根据反馈调整类似话题的出现频率

## **第四部分：项目实现建议**

### **技术架构**

```
前端（用户界面）
    ↓
话题生成引擎
    ↓
话题数据库
    ├── 基础话题库（100个种子）
    ├── 衍生话题库（动态生成）
    └── 用户话题库（用户创建）
    ↓
个性化推荐系统
    ↓
反馈与学习系统
```

### **功能模块设计**

1. **随机话题生成**：一键获取随机话题
2. **分类选择生成**：按类别获取话题
3. **智能推荐生成**：基于历史记录和偏好推荐
4. **自定义话题创建**：用户可创建并保存话题
5. **话题收藏夹**：收藏喜欢的话题
6. **对话记录**：记录聊过的话题和心得
7. **关系时间线**：按关系阶段记录话题
8. **话题分享**：分享话题给伴侣

### **用户界面设计要点**

1. **主界面**：简洁的"生成话题"按钮，显示当前话题
2. **话题卡片**：清晰显示话题内容，附带标签
3. **操作按钮**：
   - 👍 喜欢/收藏
   - 👎 跳过/不感兴趣
   - 💬 开始对话
   - 📝 记录心得
   - 🔄 重新生成
4. **设置界面**：个性化偏好设置
5. **历史界面**：查看对话历史和收藏
